## 設計原則

### アーキテクチャ
- **rooms_physical.js**: 教室の物理データ（テーブル構造）＋ `generateRoom()` 関数
- **index.html**: UI・描画・配置ロジック。起動時に `generateRoom()` で全教室のグリッドを生成
- **admin.html**: 管理者画面。教室データの閲覧・変更・追加、rooms_physical.jsのエクスポート
- 描画コードは **絶対に不用意に変更しないこと**（過去に描画コードの変更でレイアウトが破綻した経験あり）

### ROOM_LIST自動生成（Ver.2.3〜）
- index.htmlの`ROOM_LIST`は`PHYSICAL`から自動生成（パーティション子・sameAs子は除外）
- admin画面で教室を追加→エクスポートするだけでindex.htmlに自動反映
- パーティション子（I/II）は親教室のボタン横に小ボタンで表示
- sameAs教室（402→302、406→404等）は元教室とグループ表示

### 物理データ構造（rooms_physical.js）
- `tables[行][ブロック] = [テーブルサイズ配列]`
- 例: `[[3],[3,3],[3]]` = 左ブロック3脚×1台、中央ブロック3脚×2台、右ブロック3脚×1台
- 扇型教室: 1ブロック = 1つの連結机（例: `[7]` = 7脚の連結机）
- 教室タイプ: `fan`（扇型）、`rect`（長方形）、`individual`（独立机）

### 教室参照の仕組み
- **sameAs**: 同型教室の参照（例: `402`→`302`）。データは完全展開して保持（generateRoom互換）
- **parent + partitionHalf**: パーティション分割（例: `302-I`は`302`の前半5行）。データは完全展開して保持
- admin画面のエクスポートは、sameAs・パーティション子ともに完全なデータ（type, rows, sections, tables）を出力

### 着座ルール
1. **1席空け**: テーブル単位で適用。N脚テーブル → ⌊(N+1)/2⌋名
2. **寄せ方向**: 偶数ブロック(B1,B3,B5)は右寄せ、奇数ブロック(B2,B4,B6)は左寄せ
   - セクション内側の通路（①③⑤）に向かって座席を寄せる
3. **1ブロック最大5席**（201教室）: 6席になるブロックの寄せ反対側端をギャップに変換
4. 独立机（431）は1席空け不要（1机=1席）

### 201教室のブロック構造
```
壁 B1 ①通路 B2 ②通路 B3 ③通路 B4 ④通路 B5 ⑤通路 B6 壁
   ──正面左──        ──中央──        ──正面右──

着座の寄せ方向: →①← →③← →⑤←
```
- 3セクション × 2ブロック = 6ブロック、5通路
- 座席は①③⑤（セクション内側通路）に寄せて配置
- ②④（セクション間通路）側にギャップが来ることがある

### グリッド値
- `null` = 椅子なし（空間）
- `0` = ギャップ椅子（椅子はあるが着座しない）
- `1以上` = 座席番号（列優先・上から下の順で自動付与）

## 機能

### 座席配置オプション
- **詰め方**: 横（行優先）/ 縦（列優先）
- **左右**: 左から / 右から
- **並び方**: 順番（名簿順）/ 奇数→偶数 / 偶数→奇数（学籍番号末尾で判定）
- **列間隔・行間隔**: 使用する列/行の間隔
- **前/後/左/右 skip**: 使用しない行・列の数
- **ブロック内最大席数**: 各ブロックの最大座席数を調整可能（201教室はデフォルト5）
- **中央席除外チェックボックス**（201, 202）: 中央の狭い1-2人がけ席を除外
- **表示切り替え**: 学籍番号 / 学生氏名 — 座席表に表示する情報を選択

### 入出力
- **入力**: UNIPAの履修者Excelファイル（.xlsx）— 「学籍番号」「学生氏名」列を自動検出
- **出力**: PNG画像生成 → ダウンロードまたは右クリック/長押しで保存。学籍番号版と氏名版を切り替え可能（氏名版はファイル名に`_氏名`が付く）

### 管理者画面（admin.html）
- **パスワード認証**（sessionStorage）
- **教室データの閲覧・変更・追加**: 物理レイアウトのグリッド編集
- **同型教室追加**（sameAs）: 既存教室と同じ構造の教室をワンクリック追加
- **パーティション分割**: 教室を前半/後半に分割（I/II）
- **棟グループ表示**: building属性で棟ごとにサイドバー分類（デフォルト: 太陽が丘2号棟）
- **並べ替え**: 名前順/椅子数順/タイプ順
- **エクスポート**: ユーティリティ関数を含む完全なrooms_physical.jsを出力（クリップボードコピー or ファイルダウンロード）
- **トップページリンク**: ヘッダーからindex.htmlへ遷移

## 技術構成

### 依存ライブラリ（CDN）
- React 18.2.0 / ReactDOM 18.2.0
- SheetJS (xlsx) 0.18.5

### デプロイ
- `index.html` + `rooms_physical.js` の2ファイルをGitHub Pagesに配置
- `admin.html` はローカル専用（サーバにアップロード不要）
- 学生データはブラウザ内で完結（サーバー送信なし）

## ファイル構成
```
seat-assignment/
├── index.html          ← アプリ本体（UI＋描画＋配置ロジック）★GitHub Pages
├── rooms_physical.js   ← 教室物理データ＋generateRoom()　　★GitHub Pages
├── admin.html          ← 管理者画面（ローカル専用・サーバ不要）
└── README.md           ← このファイル
```

## 変更時の注意事項
- **教室の追加・変更**: admin.htmlから操作 → エクスポートでrooms_physical.jsを更新。index.htmlの編集は不要
- **同型教室の追加**: admin画面で「＋同型」ボタン。または rooms_physical.js に `sameAs` 参照＋完全データで手動追加
- **着座ルールの変更**: `index.html` 冒頭の即時実行関数（`maxPerBlock` 等）を編集
- **描画コードの変更は慎重に**: セルサイズ・座標計算・SVG生成ロジックの変更はレイアウト崩壊のリスクが高い。必ず動作中のバージョンをバックアップしてから作業すること
